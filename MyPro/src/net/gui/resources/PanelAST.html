This panel displays the Abstract Syntax Tree produced by a parser. A well-defined AST structure exists for
any rule-tree, defined by applying the following rules recursively to each node in the tree
  • A Sequence yields an "Array" (Object[]) of the sub-rule ASTs. If there is effectively one
    child node, that node's AST is returned (without the array encapsulation). An effectively empty
    Sequence produces an empty array
  • A Choice yields a 2-element array. The first element is a 0-based index of the matching
    alternative sub-rule, and the second element is the AST of the matching alternative itself
  • A RepSep yields a "List" (java.util.List) with ASTs from applying the first sub-rule.
    A RepSep with multiplicity * (0 or more) may return an empty List
  • A Literal yields a String identical to its pattern, and is shown as such in the AST
  • A Regex yields the String matching it's pattern, and is shown as "[" token-name "]"
  • A Reference yields the AST of the referred rule, and is shown as "@" rule-name
  • A node with an action function yields it's return value, and is shown as "action@" node-name
The basic structure described above is further modified as follows:
  • A multiplicity of * or + yields a "List" with ASTs from applying the basic sub-rule
  • A multiplicity of ? yields an "Option" (a 1-element array holding a (possibly null) AST
  • A multiplicity of 0 (not) or = (guard) causes the sub-rule to be dropped from the AST
    These values of multiplicity actually produce syntactic predicates
  • A drop attribute, obviously, causes the sub-rule to be ignored in the AST
